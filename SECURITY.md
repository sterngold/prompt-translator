# Security Policy

## Reporting a Vulnerability

Please do not open public issues for security problems. Use GitHub private
vulnerability reporting if it is available, or email the maintainer listed on
the profile.

This public repository is a case study. The production Worker, UI, interpreter
prompt, telemetry, and user data are intentionally private.

Include:

- A short description of the issue.
- Reproduction steps or a minimal proof of concept.
- Whether the concern affects this public case study or the private product.
